# 웹 서버 마이그레이션 가이드 (IIS & Apache)

현재 프론트엔드(HTML, CSS, JS)에서 직원 정보를 `members.json` 및 `allmember.json` 파일에 저장하고 읽어오는 구조입니다. 
기본적으로 제공해 드린 `server.js`(Node.js) 없이 **사내 범용 웹 서버(IIS 또는 Apache)**에 바로 올려서 작동시키려면, 각 서버가 지원하는 기본 백엔드 언어(C# 또는 PHP)로 JSON 파일을 처리해 주는 간단한 API 스크립트가 필요합니다.

두 가지 서버 환경에 대한 작동 방법과 필요한 추가 작업을 정리해 드립니다.

---

## 1. IIS 서버 (Windows)에서 작동하는 방법

IIS는 기본적으로 **ASP.NET (C#)** 환경을 가장 완벽하게 지원합니다.

### 필요한 추가 작업
1. **API 파일 생성 (`api.ashx`)**
   - JSON 파일을 읽고(GET) 덮어쓰는(POST) C# 코드가 담긴 `api.ashx` (Generic Handler) 파일을 웹 루트에 하나 생성합니다.
2. **프론트엔드 통신 경로 수정**
   - `script.js` 내부에 있는 서버 요청 주소를 Node.js 방식에서 IIS 방식으로 변경합니다.
   - ❌ 기존: `fetch('/api/members')`
   - ✅ 변경: `fetch('api.ashx?type=members')`
3. **폴더 쓰기 권한 부여 (가장 중요 ⭐️)**
   - 윈도우 서버에 접속하여 `assets/member/` 폴더를 우클릭 ➡️ 속성 ➡️ 보안 탭으로 이동합니다.
   - **`IIS_IUSRS`** (또는 응용 프로그램 풀 그룹) 사용자에게 **수정(Modify)** 및 **쓰기(Write)** 권한을 반드시 부여해야 합니다. (권한이 없으면 출퇴근 버튼을 눌러도 파일에 저장되지 않습니다.)

---

## 2. Apache 서버 (Linux/Windows)에서 작동하는 방법

Apache는 대부분 **PHP** 모듈과 함께 구성되어 동작하는 경우가 99%입니다.

### 필요한 추가 작업
1. **API 파일 생성 (`api.php`)**
   - `file_get_contents()`와 `file_put_contents()` 함수를 사용해 JSON 파일을 읽고 쓰는 `api.php` 파일을 웹 루트에 생성합니다.
2. **프론트엔드 통신 경로 수정**
   - `script.js` 내부에 있는 서버 요청 주소를 PHP 파일로 변경합니다.
   - ❌ 기존: `fetch('/api/members')`
   - ✅ 변경: `fetch('api.php?type=members')`
3. **폴더 쓰기 권한 부여 (가장 중요 ⭐️)**
   - 리눅스 서버 기준, 터미널에서 `assets/member/` 폴더의 소유자를 Apache 실행 계정(주로 `www-data` 또는 `apache`)으로 변경하거나 쓰기 권한(`chmod 777` 등 상황에 맞게)을 부여해야 합니다.
   - 예시 명령어: `chown -R www-data:www-data assets/member/`

---

## 요약 (어느 서버든 공통 사항)

현재 로직은 순수 HTML/JS이므로 화면은 무조건 잘 뜹니다. 다만, **데이터를 파일에 저장(출퇴근, 직원등록)하기 위해서는** 다음 두 가지만 맞춰주시면 됩니다.

1. 해당 서버 언어(IIS는 C#, Apache는 PHP)로 작성된 **파일 쓰기용 API 스크립트 1개** 추가
2. 그 스크립트가 위치한 곳을 바라보도록 **`script.js`의 주소 한 줄 변경**
3. 서버 하드디스크의 해당 JSON 저장 폴더(`assets/member/`)에 **쓰기 권한 열어주기**
